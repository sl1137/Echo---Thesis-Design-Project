# Echo Design System (Mini Version)

## 1. Product Feel

Echo should feel:
- dreamy
- soft
- breathable
- emotionally safe
- mobile-first
- calm rather than busy
- warm rather than clinical

This is not a productivity app or a technical dashboard.
It should feel like a gentle emotional support space.

---

## 2. Visual Direction

The visual style should combine:
- pastel gradients
- soft floating surfaces
- rounded corners
- low-pressure spacing
- subtle glow
- soft shadows
- minimal visual noise

Avoid:
- harsh black shadows
- overly saturated colors
- sharp corners
- cluttered layouts
- too many decorative styles at once
- mixing too many illustration styles on one screen

---

## 3. Core Color Palette

Use a restrained pastel palette.

### Primary
- Echo Purple: `#6F4BD8`

### Background Pastels
- Sky Mist: `#DDEBFA`
- Blush Pink: `#F6DDE9`
- Warm Cream: `#F7E8B8`
- Soft Lavender: `#E9E2F8`

### Neutral
- Ink: `#2E1F5E`
- Cloud White: `#FFFFFF`
- Soft Gray Border: `#E5E2EB`

### Rules
- Use Echo Purple for primary interactive elements.
- Background gradients should stay soft and low contrast.
- Avoid adding many extra accent colors unless necessary.
- Tag colors should remain muted and pastel.

---

## 4. Typography

Use simple, soft, readable typography.

### Levels
- Display: for logo / splash title
- H1: major page titles
- Body: primary text
- Caption: helper text / labels / subtle status

### Rules
- Titles should feel warm and rounded, not aggressive.
- Body text should remain highly readable.
- Avoid too many font sizes on one screen.
- Prefer bold/semibold for headings, regular/medium for body text.

---

## 5. Radius System

Use a consistent rounded language.

- Small radius: `12px`
- Medium radius: `20px`
- Large radius: `28px`
- Pill radius: `999px`

### Use
- Chips / tags / small buttons: small or pill
- Input fields / chat bubbles / cards: medium
- Major panels / floating containers / bottom nav: large

---

## 6. Shadow System

Use soft shadows only.

### Shadow S
For chips, buttons, small floating controls.

### Shadow L
For cards, main surfaces, voice orb area, bottom navigation.

### Rules
- Shadows should be blurred and subtle.
- Never use hard, heavy black shadows.
- Prefer atmospheric glow over strong drop shadow.

---

## 7. Components

### A. Primary Button
Used for:
- Start
- Continue
- Enter
- Main CTA

Style:
- rounded / pill-like
- soft shadow
- clean label
- strong enough to be clear, soft enough to feel calm

### B. Icon Button
Used for:
- back
- menu
- microphone
- close
- secondary action

Style:
- circular
- soft fill
- centered icon
- consistent sizing

### C. Chat Bubble
Used only in text mode.

User bubble:
- stronger purple or tinted primary
- right aligned

Echo bubble:
- soft lavender or light neutral
- left aligned

Rules:
- readable spacing
- generous padding
- not too narrow
- soft rounded corners

### D. Voice Surface
Used in voice mode.

Should include:
- central orb / active focus object
- breathing space
- minimal text or no transcript
- microphone / stop controls
- audio-only emphasis

Rules:
- voice mode should feel calm, immersive, and uncluttered
- do not show text transcript during live voice conversation

### E. Bottom Navigation
Used for the larger app prototype.

Style:
- floating rounded container
- soft shadow
- 3–4 clear tabs max

---

## 8. Mode Rules

### Text Mode
- show message list
- show text input
- show send button
- user and Echo communicate via visible text

### Voice Mode
- no transcript on screen
- no visible AI text reply
- use large breathing space and voice state UI
- AI responds with voice only
- keep interface minimal

---

## 9. Layout Rules

- Always design mobile-first.
- Keep generous vertical spacing.
- Use clear hierarchy.
- Let the main interaction be visually dominant.
- Decorative background should never overpower interaction.

---

## 10. Illustration Rules

Illustrations are allowed, but must be controlled.

Allowed:
- soft dreamy island environments
- clouds
- floating orb
- one recurring mascot / companion style

Avoid:
- mixing too many different illustration aesthetics
- adding decorative stickers without purpose
- letting illustrations compete with the main CTA

---

## 11. What to Prioritize

When implementing screens, prioritize:
1. clarity of interaction
2. emotional tone
3. consistency of surfaces
4. readability
5. beauty

Beauty should support emotional safety, not overwhelm usability.
